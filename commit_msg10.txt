Remove misleading "Confirmed" section header from Social's Coordinator attendee list

Iain (2026-07-14): the green "CONFIRMED" heading above the attendee list on
a paid Social event implied every row under it had actually paid, when in
reality confirmed just means seat-confirmed -- payment is tracked
separately per row (see the "hivetest" row on Bastille Day: confirmed seat,
still Unpaid). Green as a header colour also reads as a pass/status signal,
not a neutral label, compounding the confusion.

Removed the header entirely rather than relabel it -- each row already
carries its own Paid/Unpaid state via the canonical bookingStatusBadge
colours, so a blanket heading above the whole group was redundant at best
and misleading at worst. Left the "Waitlist" header as-is: it's the rarer
case worth flagging, and it doesn't carry the same "all paid" implication.

Movies and Book Club's Coordinator views (components/EventSlideOut.js) never
had this header at all -- Social was the only hub with it, so this isn't a
cross-hub consistency fix, just removing something that shouldn't have been
there in the first place.

Verified: npm run build clean.
