Fix Cancel Booking feeling unresponsive -- no loading state, blocking push

Reported: Cancel confirm was "extremely slow" and needed multiple clicks to
respond.

Found two real, verified issues (checked code, not guessed):

1. ConfirmDialog's "Yes, cancel" / "Keep it" buttons had no disabled/loading
   state at all -- unlike every other action button in this file (Book Now,
   Modify, Mark Submitted, Add Walk-up Booking all correctly disable while
   their request is in flight). Whether the request took 200ms or 3s, the
   button gave zero feedback, so tapping again looked like the obvious thing
   to do -- which then fires a second overlapping cancel_booking request
   against the same booking. Fixed: both the EC/admin cancel path
   (CoordinatorPanel's cancelBooking) and the resident's own self-cancel path
   (handleCancel) now track an in-flight state, disable both dialog buttons
   and relabel to "Cancelling..." while a request is running, and ignore a
   repeat tap instead of firing a second request.

2. lib/notify.js's push send was awaited as part of notify(), which every
   cancel_booking call (and promoteWaitlist's per-waitlisted-seat loop) waits
   on before responding. sendPushToMember() already treats push as
   best-effort internally (catches its own errors) -- there was nothing to
   gain by blocking the HTTP response on a network round-trip to Apple/
   Google/Mozilla's push service per subscription. Made it fire-and-forget.
   Push isn't live in production yet (VAPID env vars not yet added to
   Vercel), so this isn't today's bottleneck, but it would have become one
   the moment push goes live and an event has more than one push-subscribed
   attendee -- fixed now while touching this exact code path rather than
   waiting to rediscover it later.

Likely also a factor, not fixable in code: Vercel serverless cold starts add
real latency to an infrequently-hit route's first request after idle, which
matches "may just have been slow on the first attempt." Can't verify this
one directly from here.

Verified: npm run build clean.
